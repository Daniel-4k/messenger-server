# Связь — мессенджер с реальным сервером

> ⚠️ **Самое важное правило этого проекта:** никогда не открывайте
> `public/index.html` двойным кликом в файловом менеджере. Браузер
> при открытии файла напрямую (адрес вида `file://...`) блокирует
> подключение `script.js` — из-за этого ничего не будет работать,
> включая кнопку «Продолжить». Приложение нужно **всегда** открывать
> через запущенный сервер — то есть сначала `npm start`, а затем
> адрес вида `http://localhost:3000` в браузере. Если вы всё же
> откроете файл напрямую, приложение покажет об этом предупреждение
> прямо на экране.

Это рабочий мессенджер из двух частей:

- **server.js** — backend на Node.js: регистрация по номеру телефона, поиск
  пользователей, хранение чатов, фото и голосовых сообщений, обмен в
  реальном времени через WebSocket, сигналинг для аудиозвонков.
- **public/** — клиент (то, что открывается в браузере): тот же дизайн,
  что был раньше, но теперь данные реально передаются между разными
  людьми, а не лежат только в вашем браузере.

Самое важное отличие от предыдущей версии: **теперь если ты дашь ссылку
другу, вы действительно сможете переписываться и звонить друг другу.**

---

## 1. Запуск на своём компьютере (для проверки)

Нужен установленный [Node.js](https://nodejs.org) версии 18 или новее.

```bash
cd messenger-server
npm install
npm start
```

После этого сервер будет работать на `http://localhost:3000`.
Откройте этот адрес в браузере — увидите экран приветствия.

**Чтобы протестировать переписку между двумя «людьми» прямо сейчас:**
откройте `http://localhost:3000` в обычном окне браузера и ещё раз —
в окне в режиме инкогнито (или в другом браузере). Зарегистрируйте два
разных номера телефона — и пишите от одного аккаунта другому.

## 2. Как зарегистрироваться (демо-режим SMS)

SMS реально не отправляются — это стоит денег и требует отдельного
провайдера (см. раздел про SMS ниже). Вместо этого:
- Код подтверждения всегда **1234**
- Либо сервер выводит код в консоль и во всплывающем уведомлении в самом
  интерфейсе, чтобы не лезть в логи

## 3. Как дать ссылку друзьям, чтобы они могли писать вам

Сервер должен быть доступен из интернета — то есть либо

**а) запущен на хостинге** (рекомендуется), либо
**б) ваш компьютер виден извне** (сложнее, не рекомендуется для постоянного использования).

### Вариант А — бесплатный хостинг (проще всего)

Подойдут сервисы, которые запускают Node.js-проекты по git-репозиторию:
**Render.com**, **Railway.app**, **Fly.io** — у всех есть бесплатный
тариф, достаточный для теста с друзьями.

Общий план для любого из них:
1. Залить эту папку (`messenger-server`) в репозиторий на GitHub.
2. Подключить репозиторий к выбранному хостингу.
3. Указать команду запуска: `npm install && npm start`.
4. Хостинг выдаст вам адрес вида `https://ваш-проект.onrender.com`.
5. Этот адрес и есть ссылка, которую можно скидывать друзьям —
   они открывают её в браузере на телефоне или компьютере.

⚠️ **Важно про аудиозвонки**: браузеры разрешают доступ к микрофону
(`getUserMedia`) только на `https://` или `http://localhost`. На обычном
`http://` с внешним IP звонки и запись голосовых сообщений работать не
будут. Бесплатные хостинги (Render, Railway и т.п.) сами выдают `https://`
из коробки — это ещё одна причина использовать их вместо «голого» сервера.

### Вариант Б — в локальной сети (быстро, но только рядом)

Если друзья физически в одном Wi-Fi с вами:
1. Узнайте свой локальный IP: `ipconfig` (Windows) или `ifconfig`/`ip a` (Mac/Linux).
2. Запустите сервер: `npm start`.
3. Друзья открывают `http://ВАШ_IP:3000` в браузере, будучи в той же сети.
4. Звонки и запись голоса по обычному `http://` в локальной сети чаще
   всего тоже не сработают (требование браузера про https) — для этого
   варианта лучше ограничиться текстом и фото, либо настроить локальный
   HTTPS-сертификат (сложнее, не описано здесь).

## 4. Подключение настоящих SMS (когда будете готовы к реальному запуску)

Сейчас в `server.js` есть функция `/api/auth/request-code`, где код
подтверждения просто генерируется на сервере (`"1234"`). Чтобы код
реально прилетал на телефон по SMS, нужно:

1. Зарегистрироваться у SMS-провайдера, например **Twilio**,
   **SMS.ru**, **Beeline B2B** или похожего сервиса.
2. В функции `request-code` вместо строки `const code = "1234"` вызвать
   API провайдера для отправки SMS с этим кодом на номер `phone`.
3. Это платная услуга (обычно несколько рублей за SMS) — провайдер
   выдаст вам API-ключ и документацию по отправке.

## 5. Что использовать вместо простого JSON-файла, если людей станет много

Сейчас вся база данных — это файл `data/db.json`, который пересохраняется
при каждом изменении. Для компании друзей (десятки человек) этого более
чем достаточно. Если вырастет до тысяч активных пользователей, стоит
перейти на настоящую базу данных (PostgreSQL, MySQL) — структура кода
в `db.js` сделана так, чтобы это можно было заменить без переписывания
всего сервера.

## 6. Как работают звонки технически

Звонок устроен на технологии **WebRTC**:
- Сервер передаёт только служебные «записки» — кто кому звонит, согласие
  на разговор, и техническую информацию для соединения (SDP/ICE).
- Сам голос идёт **напрямую** между двумя браузерами (peer-to-peer), без
  прохождения через сервер — это быстрее и снимает нагрузку с сервера.
- Для соединения через NAT используется публичный STUN-сервер Google.
  В сложных сетях (некоторые мобильные операторы, корпоративные сети)
  для надёжности может понадобиться ещё и TURN-сервер — это отдельная
  настройка, выходящая за рамки базовой версии.

## Структура проекта

```
messenger-server/
├── server.js        — HTTP API + WebSocket сервер
├── db.js             — простое файловое хранилище данных
├── package.json
├── data/
│   ├── db.json        — сама база (создаётся автоматически)
│   └── uploads/        — загруженные фото и голосовые сообщения
└── public/
    ├── index.html       — разметка и стили интерфейса
    └── script.js          — вся логика клиента (работа с API, WebSocket, WebRTC)
```
